# 素材整理指南

Use this when the user wants to train, learn, or update a personal writing style from their own material.

## Recommended folder layout

```text
writing-style/
├── samples/
│   ├── official-materials/
│   ├── public-account/
│   ├── speeches/
│   └── xiaohongshu/
├── edits/
│   ├── 2026-01-ai-draft.md
│   └── 2026-01-user-edited.md
├── style-profile.md
└── forbidden-patterns.md
```

## File naming

Use names that preserve context:

```text
2026-03-15_public-account_ai-tools-review.md
2026-04-02_speech_work-deployment.md
2026-05-08_official-summary_quarterly-work.md
```

## Minimum sample guidance

- 3 samples: rough starter profile.
- 5-10 samples: useful genre profile.
- 10-20 samples: stable rules and reusable structures.
- Before/after edits: high value even with only 2-3 pairs.

## What to preserve

- Title and final published title if different.
- Target reader and platform.
- Original prompt or task background when available.
- Whether the piece was self-written, AI-assisted, or heavily edited.
- Final version and rejected draft when available.

## What to avoid

- Mixing genres into one profile too early.
- Including private third-party writing without permission.
- Treating one viral piece as the whole style.
- Removing all "messy" drafts; edits reveal preference.
