# Drafting Official Materials

## Overview

Draft formal Chinese official materials by first identifying the document type, then building a policy-grounded structure, drafting in restrained institutional language, and checking format, logic, facts, and tone.

This skill supports two related but different categories:

- **法定公文**: 通知、通报、请示、报告、函、会议纪要等，where format and institutional relationship matter.
- **机关材料**: 讲话稿、汇报材料、调研报告、工作总结、信息简报、经验材料等，where structure, stance, policy language, and material organization matter.

## Workflow

1. **Clarify the task boundary**
   - Identify the document type, recipient, sender role, purpose, urgency, length, and whether the user has supplied facts that must be preserved.
   - If critical facts are missing, ask only for what blocks drafting. Otherwise draft with explicit placeholders such as `[请补充时间]`.

2. **Select the writing pattern**
   - For statutory documents, use `references/document-types.md`.
   - For speeches, summaries, reports, briefings, and research materials, use `references/material-patterns.md`.
   - For final review, use `references/quality-checklist.md`.

3. **Build the outline before prose**
   - State the main purpose in one sentence.
   - Arrange content by `背景/依据 -> 主要事项 -> 工作要求 -> 保障措施` or the pattern for the selected genre.
   - Keep hierarchy balanced. Avoid a first-level point with no matching peer.

4. **Draft in official style**
   - Use precise subjects: `各单位`, `有关部门`, `我局`, `全市系统`, `各基层党组织`.
   - Prefer verbs of governance and implementation: `贯彻落实`, `统筹推进`, `压实责任`, `健全机制`, `加强督导`, `确保实效`.
   - Avoid slogans without substance. Tie every claim to a measure, object, deadline, indicator, or responsible party when available.

5. **Check before final answer**
   - Confirm the document type matches the request.
   - Check political stance, institutional relationship, legal/formal tone, internal logic, factual placeholders, duplicated phrasing, and closing formula.
   - For `.docx` formatting or GB/T layout work, use a document formatting skill/tool if available.

## Drafting Rules

- Preserve user-provided facts, names, dates, data, and organizational relationships. Do not invent policy citations, leaders' names, meeting names, statistics, or legal bases.
- If the user asks for party/government content involving current policies, laws, leaders, or recent events and exactness matters, verify with current sources before asserting.
- Use concise paragraphs. For official documents, each paragraph should advance a function: basis, decision, requirement, responsibility, deadline, or closing.
- Use measured language. Do not write marketing copy, internet slang, exaggerated praise, or emotional appeals unless the user explicitly asks for a less formal draft.
- Keep titles literal and functional. Avoid clever, vague, or sensational titles.

## Quick Type Selection

| User asks for | Use this type | Key output |
|---|---|---|
| 下发任务、安排工作 | 通知 | What to do, who does it, deadline, requirements |
| 向上级说明情况但不请求批复 | 报告 | Situation, work done, issues, next steps |
| 向上级请求批准或支持 | 请示 | One request, reasons, proposed solution, closing request |
| 表扬、批评、告知重要情况 | 通报 | Facts, assessment, decision or lessons, requirements |
| 平级或不相隶属单位沟通 | 函 | Matter, basis, requested coordination, reply expectation |
| 会议结果和议定事项 | 会议纪要 | Meeting context, agreed matters, responsibilities |
| 领导发言 | 讲话稿 | Situation judgment, key tasks, requirements, mobilization |
| 阶段性复盘 | 工作总结 | Achievements, measures, problems, next plan |
| 调查分析 | 调研报告 | Background, findings, problems, causes, recommendations |
| 信息上报/内部动态 | 信息简报 | Lead, facts, measures, effect, next step |

## Common Mistakes

- Mixing `请示` and `报告`: use `请示` only when requesting approval; use `报告` when informing.
- Writing a beautiful but empty draft: replace abstract phrases with tasks, mechanisms, people, timelines, data, and deliverables.
- Overusing four-character clusters: use them for rhythm, not as filler.
- Ignoring institutional hierarchy: recipient and sender relationship determines wording, request strength, and closing.
- Adding fabricated policy basis: use placeholders or ask for the exact source.

## Output

When the user asks for a draft, provide:

1. A polished draft ready for review.
2. A short note listing assumptions or placeholders.
3. Optional revision suggestions when the supplied material is thin.
